Consultas al Equipo SAP

Integración Despachos Petroperú — Servicio Web ↔ SAP (Interfaces 3.1 y 3.2) — Mayo 2026

1. Interfaz de Planificación de Carga (3.1) — SAP → Servicio Web

#Pregunta
1¿URL base de SAP (dev / qa / prod) desde donde enviarán el POST a /planificacion? ¿Es a través de SAP PI/PO o directo desde S/4HANA?
2¿Qué valor de X-API-Key usará SAP para autenticarse? Necesitamos el valor por cada ambiente.
3¿SAP soporta el comportamiento de idempotencia? Si reenvían un mismo NroTransporte en estado Pendiente, hacemos update (delete + re-insert). En cualquier otro estado respondemos 409 Conflict. ¿Les funciona ese comportamiento?
4Confirmar formato de FechaCarga: ¿siempre yyyyMMdd sin hora? ¿En qué zona horaria?
5Producto: ¿es código de material SAP? ¿Existe catálogo fijo o debemos validar contra el maestro de materiales?
6UMVol: ¿qué códigos de unidad de medida usa SAP (ej. GAL, L, BBL)? ¿Catálogo disponible?
7API en el Detail: ¿proviene del maestro de materiales de SAP o es un valor calculado por ellos? ¿Es obligatorio?
8NroEntrega: ¿representa un delivery document, sales order, o shipment en SAP?
9BayQueuePriority: ¿qué valores posibles y qué criterio usa SAP para asignar la prioridad de cola en bahía?
10IndViaje: ¿qué significa este indicador y cuáles son sus valores posibles?
11CustomerCode, Destinatario, SCOP: ¿qué representan en SAP y qué longitud real manejan?
12¿Hay límite máximo de compartimentos por transporte? (El servicio valida que NroCompartimento no se duplique dentro de la orden.)

2. Interfaz de Confirmación de Carga (3.2) — Servicio Web → SAP

#Pregunta
13URL exacta del endpoint SAP que recibirá el POST con el XML de confirmación (por ambiente: dev, qa, prod).
14¿Qué valor de X-API-Key espera SAP en el request de confirmación? (Puede ser distinto al de la interfaz 3.1.)
15Confirmar que el schema XML de salida es correcto (ver anexo al final). ¿Requieren campos adicionales? ¿El campo NroTransporte dentro de cada Detail es necesario o basta con el del Header?
16Formato de decimales: ¿SAP espera punto (.) o coma (,) como separador decimal? Confirmar precisión: Temperatura (6,2), APIDespachado (8,4), VolObservado (16,2), Vol60 (16,2).
17¿Qué HTTP status codes devuelve SAP ante una confirmación? Asumimos: 2xx = éxito, 4xx = error no reintentable, 5xx = reintento. ¿Confirman?
18Estrategia de reintentos actual: 3 intentos máximos con backoff 10s / 30s / 60s. Si los 3 fallan, marcamos Error. ¿Les parece adecuado? ¿Tienen ventana de timeout en SAP?
19¿Qué proceso de negocio dispara SAP al recibir la confirmación? (actualiza delivery, goods issue, facturación, etc.)
20¿Existe un procedimiento de conciliación en SAP cuando una confirmación queda en estado Error? ¿O se maneja manualmente por operaciones?
